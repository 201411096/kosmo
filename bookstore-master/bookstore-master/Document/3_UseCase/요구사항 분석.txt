1. 사용자 요구사항 도출

1-1. 사용자 타겟 설정
   - 책은 읽고 싶으나 뭐부터 읽어야 할지 막막한 사람들
    - 체계적인 독서습관으로 점진적인 독서량 증가 및 독서패턴 형성을 원하는 사람들
   - 책의 홍보가 필요한 사람들
   - 도서정가제로 인해 도서 구매비용이 부담스러운 사람들
   - 한 권을 읽더라도 효율적인 독서를 원하는 사람들

1-2. 요구사항 정의
(1) 사이트 기본 기능
- 베스트셀러, 새로 나온 책을 보여주는 기능
- 팝업창에 베스트셀러 1위 보여주는 기능

(2) 도서구매 서비스
- 상품 리뷰, 문의하기 게시글 기능
- 장바구니 기능

(3) 도서대여 서비스
- 도서대여 시 수령일, 반납일, 대여료를 보여주는 기능
- 대여일 연장하는 기능
- 대여가능, 대여불가 등 대여현황을 확인하는 기능

(4) 데이터 분석 서비스
- 사용자 구매이력에 따른 독서유형을 분석하는 기능
- 데이터 분석을 바탕으로 유사한 장르 및 작가의 상품을 추천하는 기능
- SNS를 연동하여 독서리뷰, 데이터 분석을 공유함으로써 독서량 및 홍보를 향상하는 기능 

(5) 지점 안내 서비스
- 오프라인 지점 정보 확인하는 기능
- 지도정보 기능

2. 웹사이트 컨셉 및 전략
2-1. 개발 서비스의 우수성
- 독서패턴 분석으로 인한 독서 욕구 향상
- 도서 구매가 아닌 대여를 통해 폭넓은 도서상품 접근
- 사이트 하나로 구매, 정보 공유, 도서패턴을 파악하여 고객편리 제공

2-2. 기대효과
- 도서비용 감소
- 독서문화 확산
